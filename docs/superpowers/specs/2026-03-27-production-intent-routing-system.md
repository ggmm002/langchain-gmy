# 企业 Agent 意图识别与路由系统设计说明

## 目标

围绕企业级 Agent 系统构建一条稳定的意图识别与路由主链路，把用户查询准确分发到以下子系统：

- 企业私域知识库 RAG 检索
- 企业供应量数据查询分析报告
- 供应量工作流自动化执行

## 设计原则

- 生产容错优先：即使 LLM 不可用，也能通过规则引擎完成稳定路由
- 明确可审计：每一步都有结构化 trace，方便排错和后续接入监控
- 路由先于执行：对高风险工作流必须设置确认与 dry-run 护栏
- 前后端同构：后端输出的分类结果、风险标签、缺失槽位和轨迹可以直接驱动前端展示

## 路由链路

使用 LangGraph 将路由流程编排为固定模板：

1. `prepare`
2. `rules`
3. `llm`
4. `fusion`
5. `knowledge_rag | supply_analytics | workflow_automation | clarification`

## 意图定义

### knowledge_rag

用户需要查询企业知识库、制度、SOP、FAQ、流程说明等文档型内容。

### supply_analytics

用户需要查询供应量、交付量、达成率、趋势波动，或要求产出分析报告。

### workflow_automation

用户需要触发、执行、启动、同步、通知某个业务流程或自动化任务。

### clarification

用户请求存在严重歧义、跨域冲突或关键槽位缺失，当前不适合直接路由。

## 生产级能力点

- 规则分类器负责首轮稳定性
- LLM 结构化分类器负责补充复杂语义判断
- 融合策略负责处理规则与 LLM 分歧
- 高风险执行请求默认走确认护栏
- 子系统输出统一封装为 `SubsystemResult`

## 前端要求

- 展示最终识别结果
- 展示规则判定、LLM 判定、融合结果
- 展示 trace 时间线
- 展示子系统返回数据
- 支持 dry-run 和执行开关
